# Feature Specification: 作业/毕设需求管理系统（JT-Hub）v2

**Feature Branch**: `001-jthub-system`  
**Created**: 2026-04-19 | **Updated**: 2026-04-23 v3（移除微信小程序端）  
**Status**: Clarified ✅ v3

---

## 项目概述

构建一个"作业/毕设需求对接与进度追踪系统"，用户通过 **PC 网页端**提交需求，管理员通过**独立后台**管理订单、类型定价与活动公告。

- **PC 网页端**：主要用户入口，浏览器访问，登录后提交需求、查看订单进度
- **管理后台**：管理员专属，订单管理 + 类型定价 + 活动管理 + 数据统计
- **后端**：Node.js + Fastify RESTful API，PostgreSQL 数据库

---

## 核心业务流程

```
用户进入 PC 网页
    → 首页（展示活动公告 + 价格表 + 提交入口）
    → 点击「提交需求」→ 输入微信号登录（获取身份）
    → 填写需求表单（课程名称 / 需求类型 / 年级 / 截止日期 / 联系微信号）
    → 提交成功页（显示订单号 + 提示「添加 Jt--04 备注订单号」）
    → 同步到「我的订单」页面

管理员收到 Server酱 推送 → 打开管理后台 → 查看订单 → 修改状态（接单/进行中/已完成）
    → 用户在「我的订单」→「订单详情」查看最新进度
```

## User Stories

### Story 1 — 首页浏览（Priority: P1）

用户打开 PC 网页，看到首页：活动公告置顶、价格表展示（各需求类型报价）、快捷进入「提交需求」按钮。

**Acceptance Scenarios**:
1. **Given** 用户打开首页，**When** 页面加载，**Then** 展示最新活动公告列表（折扣+通知混合按时间倒序）、需求类型价格表、「立即提交需求」按钮；数据每60秒自动静默刷新
2. **Given** 管理员未发布活动，**When** 首页加载，**Then** 显示「暂无活动」占位文案
3. **Given** 用户点击价格表中的类型，**When** 点击，**Then** 直接跳转提交需求页并预选该类型

---

### Story 2 — 用户提交需求（Priority: P1）

用户填写需求表单，提交后获得唯一订单号，页面引导添加管理员微信 `Jt--04`。

**Acceptance Scenarios**:
1. **Given** 用户在提交页，**When** 填写所有必填项并提交，**Then** 生成唯一订单号（如 `JT-20260419-A3F2`），跳转成功页
2. **Given** 提交成功页，**When** 页面加载，**Then** 显示订单号 + 「请添加微信 Jt--04，备注以上订单号」+ 一键复制微信号按钮
3. **Given** 订单提交成功，**When** 数据库写入完成，**Then** Server酱 5秒内向管理员推送微信消息（含订单号/课程名/联系微信）
4. **Given** 用户提交后，**When** 进入「我的订单」，**Then** 订单已出现在列表中（状态：待确认）
5. **Given** 表单必填项为空，**When** 点击提交，**Then** 前端校验拦截，提示具体缺失字段

---

### Story 3 — 我的订单（Priority: P1）

用户在「我的订单」页查看所有历史订单，可进入详情页查看进度。

**Acceptance Scenarios**:
1. **Given** 已登录用户，**When** 进入「我的订单」，**Then** 展示该用户所有订单，按时间倒序，显示状态徽标
2. **Given** 用户点击订单卡片，**When** 跳转详情，**Then** 展示订单完整信息（课程/类型/年级/截止/微信/状态/报价）；管理员内部备注不对用户展示
3. **Given** 用户未登录，**When** 访问需登录页面，**Then** 跳回首页并提示登录

---

### Story 4 — 活动信息（Priority: P2）

用户在「活动公告」页查看折扣活动与公告通知。

**Acceptance Scenarios**:
1. **Given** 用户进入活动页，**When** 页面加载，**Then** 展示所有活动（折扣+公告混合，按时间倒序）；每60秒自动静默刷新
2. **Given** 活动已过期，**When** 展示，**Then** 显示「已结束」标签但仍展示内容
3. **Given** 活动有倒计时，**When** 展示，**Then** 显示距截止时间剩余天数

---

### Story 5 — 管理员接单与状态更新（Priority: P1）

管理员通过后台查看订单、修改状态、填写报价和备注。

**状态流转**:
```
PENDING（待确认）→ ACCEPTED（已接单）→ IN_PROGRESS（进行中）→ COMPLETED（已完成）
任意状态 → CLOSED（已关闭，终态）
```

**Acceptance Scenarios**:
1. **Given** 管理员登录后台，**When** 进入订单列表，**Then** 展示所有订单，支持按状态筛选和关键词搜索，顶部显示各状态汇总数量
2. **Given** 管理员打开订单详情，**When** 修改状态为「已接单」，**Then** 状态更新，Server酱 向管理员推送状态变更消息
3. **Given** 管理员填写报价，**When** 保存，**Then** 用户在订单详情可看到报价

---

### Story 6 — 需求类型与定价管理（Priority: P2）

管理员在后台增删改需求类型，每种类型设定参考价格，用户提交时从下拉框选择。

**默认类型**:
- 期中内作业
- 期末作业  
- 毕业设计

**Acceptance Scenarios**:
1. **Given** 管理员在类型管理页，**When** 新增类型并设价格，**Then** PC 提交表单的下拉框实时更新
2. **Given** 管理员修改某类型价格，**When** 保存，**Then** 首页价格表60秒内同步更新
3. **Given** 管理员停用某类型，**When** 操作，**Then** 该类型从用户侧下拉框隐藏，历史订单不受影响

---

### Story 7 — 活动管理（Priority: P2）

管理员发布折扣活动和公告通知，前端实时展示。

**Acceptance Scenarios**:
1. **Given** 管理员发布活动，**When** 设置标题/内容/类型/起止时间，**Then** 前端首页和活动页60秒内展示
2. **Given** 管理员删除活动，**When** 操作，**Then** 前端60秒内下架

---

### Story 8 — Server酱推送（Priority: P2）

新订单创建时，系统自动向管理员推送微信消息通知。

**推送触发时机**:
- 新订单创建 → 推送给管理员

**推送内容**:
```
📋 JT-Hub 新订单
订单号：JT-20260419-A3F2
课程：高等数学期末作业
类型：期末作业
年级：大三
联系微信：xxxxxxxxx
截止：2026-06-30
```

## 需求表单字段规范

| 字段 | 类型 | 说明 |
|---|---|---|
| 课程名称 | 文本输入 | 必填，如「高等数学」「毕业设计」 |
| 需求类型 | 下拉选择 | 必填，从管理后台动态获取（含参考价格） |
| 年级 | 下拉选择 | 必填：大一 / 大二 / 大三 |
| 作业截止日期 | 日期选择器 | 必填，须为未来日期 |
| 联系微信号 | 文本输入 | 必填，用于管理员联系用户 |

> 不再需要用户估价；不展示收款码；无邮箱字段；无详细描述字段

## 页面结构

### PC 网页端

| 页面 | 路由 | 说明 |
|---|---|---|
| 首页 | `/` | 活动公告 + 价格表（可点击预选）+ 提交入口 |
| 活动公告 | `/activity` | 活动公告列表（含已结束标签/剩余天数） |
| 我的订单 | `/my-orders` | 我的订单（需登录） |
| 提交需求 | `/submit` | 需求表单，支持 typeId 预选参数 |
| 提交成功 | `/result` | 订单号 + 引导添加微信 |
| 订单详情 | `/orders/:id` | 订单详情 + statusHistory 时间线 |

### 管理后台

| 页面 | 路由 | 说明 |
|---|---|---|
| 登录 | `/admin/login` | 管理员账号密码 |
| 数据统计 | `/admin/` | 今日/本周统计 + 各状态分布 + 最近5条订单 |
| 订单列表 | `/admin/orders` | 全部订单，状态汇总条 + 筛选+搜索 |
| 订单详情 | `/admin/orders/:id` | 查看+改状态+填报价+写备注+时间线 |
| 需求类型管理 | `/admin/order-types` | 增删改类型+定价 |
| 活动管理 | `/admin/activities` | 增删改活动/公告 |

## 移除的功能

| 功能 | 移除原因 |
|---|---|
| 微信小程序 | 改为纯 PC 网页端，降低维护复杂度 |
| 文件上传/下载 | 改为线下通过微信传文件 |
| 短链接生成 | 随文件上传一起移除 |
| 邮件通知（Resend） | 用户无邮箱字段，改用微信沟通 |
| 收款码展示 | 改为添加微信私聊定价 |
| Web 端匿名查询 | 改为登录后查询 |
| 预估价字段 | 由管理员统一定价 |

## Key Entities

- **OrderType（需求类型）**: 管理员定义，含名称、描述、参考价格、是否启用
- **Activity（活动/公告）**: 管理员发布，含标题、内容、类型（折扣/通知）、起止时间
- **Order（订单）**: 课程名称、需求类型、年级、截止时间、联系微信、状态、报价、备注
- **User（用户）**: 微信号 + JWT 认证
- **StatusHistory（状态历史）**: 记录每次状态变更
- **Notification（通知日志）**: 记录 Server酱 推送结果

---

## Success Criteria

- **SC-001**: 用户从打开页面到成功提交需求，全程不超过 2 分钟
- **SC-002**: 管理员在新订单创建后 1 分钟内收到 Server酱 微信推送
- **SC-003**: 首页价格表与管理后台定价60秒内同步（轮询刷新）
- **SC-004**: 系统支持每日 100+ 订单处理不出现性能瓶颈

---

## Assumptions

- 初期单管理员运营，管理员微信号固定为 `Jt--04`（环境变量配置）
- 需求类型初始3种（期中内作业/期末作业/毕业设计），管理员可随时增删改
- Server酱 Token 由管理员提供，作为环境变量注入
- PC 端用户输入微信号登录（虚拟 code 方式，开发模式兜底）
