**Feature Branch**: `001-jthub-system`  
**Created**: 2026-04-19 | **Updated**: 2026-04-28 v4（全面 H5 改版 + 完整注册登录 + 论坛模块）  
**Status**: Clarified ✅ v4

## 项目概述

构建一个"作业/毕设需求对接与进度追踪系统"，用户通过 **PC/手机网页端（响应式 H5）** 注册登录后提交需求，管理员通过**独立深色后台**管理订单、内容发布与数据统计。

- **PC/H5 用户端**：响应式设计，浅色现代风格，顶部导航 + 数据卡片，支持移动端
- **管理后台**：深色主题（仿雨云风格），左侧导航栏 + 卡片数据面板
- **后端**：Node.js + Fastify RESTful API，PostgreSQL 数据库

---

## 核心业务流程

```
用户进入首页（响应式 H5）
    → 首页：顶部导航 + 历代完成作业轮播 + 活动公告 + 价格表
    → 点击「提交需求」→ 未登录则跳转注册/登录页
    → 注册：用户名/昵称/邮箱/密码/手机/微信号/年级 → 邮箱验证码确认
    → 登录：邮箱或用户名 + 密码
    → 填写需求表单 → 提交成功 → 跳转「我的订单」
    → 论坛/公告区：浏览管理员发布的文档/公告，发表评论

管理员收到 Server酱 推送 → 打开管理后台（深色面板）
    → 查看订单 → 修改状态/报价/备注
    → 发布轮播图/历代作品 → 发布论坛文档 → 管理活动公告
```

## User Stories

### Story 1 — 首页浏览（Priority: P1）

**Acceptance Scenarios**:
1. **Given** 用户打开首页，**When** 页面加载，**Then** 展示：顶部 Logo + 导航 + 用户信息区、历代完成作业轮播图（图片+课程名+完成时间+好评语）、活动公告列表、需求类型价格表；数据60秒静默刷新
2. **Given** 管理员未发布轮播，**When** 首页加载，**Then** 轮播区显示默认占位图
3. **Given** 用户在手机端访问，**When** 加载任意页面，**Then** 布局自动适配手机屏幕，导航收起为汉堡菜单

---

### Story 2 — 用户注册（Priority: P1）

用户填写注册表单，邮箱收到验证码后完成注册。

**注册字段**:
| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 用户名 | 文本 | ✅ | 唯一，字母数字下划线，4-20字符 |
| 昵称 | 文本 | ✅ | 显示名称，2-20字符 |
| 邮箱 | 邮箱 | ✅ | 唯一，用于接收验证码 |
| 密码 | 密码 | ✅ | 至少8位，含字母和数字 |
| 手机号 | 文本 | ✅ | 11位手机号 |
| 微信号 | 文本 | ✅ | 用于管理员联系，提交订单时自动带入 |
| 年级 | 下拉 | ✅ | 大一 / 大二 / 大三 |

**Acceptance Scenarios**:
1. **Given** 用户填写所有必填项，**When** 点击「发送验证码」，**Then** 系统向邮箱发送6位验证码（5分钟有效）
2. **Given** 用户输入正确验证码，**When** 点击「完成注册」，**Then** 账户创建成功，自动登录跳转首页
3. **Given** 用户名或邮箱已存在，**When** 提交注册，**Then** 提示具体重复字段
4. **Given** 验证码过期，**When** 用户提交，**Then** 提示重新发送，60秒冷却期

---

### Story 3 — 用户登录（Priority: P1）

**Acceptance Scenarios**:
1. **Given** 用户输入邮箱/用户名 + 密码，**When** 点击登录，**Then** 验证成功后 JWT 存入本地，跳转上一页
2. **Given** 密码错误，**When** 登录，**Then** 提示「账号或密码错误」，连续5次失败锁定5分钟
3. **Given** 用户已登录，**When** 打开登录页，**Then** 自动跳转首页
4. **Given** 用户忘记密码，**When** 点击「忘记密码」，**Then** 跳转密码重置页面，输入邮箱接收验证码，验证后设置新密码

---

### Story 3b — 忘记密码（Priority: P1）

**Acceptance Scenarios**:
1. **Given** 用户在登录页点击「忘记密码」，**When** 输入注册邮箱并提交，**Then** 系统发送重置验证码（10分钟有效），不泄露邮箱是否已注册
2. **Given** 用户输入正确验证码和新密码，**When** 提交，**Then** 密码重置成功，跳转登录页
3. **Given** 验证码过期或错误，**When** 提交，**Then** 提示重新发送，60秒冷却期

---

### Story 4 — 我的账户（Priority: P2）

**Acceptance Scenarios**:
1. **Given** 已登录用户，**When** 点击顶部头像，**Then** 展示个人信息（昵称/邮箱/微信/年级）+ 可编辑
2. **Given** 用户修改密码，**When** 输入旧密码 + 新密码，**Then** 更新成功并重新登录
3. **Given** 用户邮箱未验证，**When** 在个人中心点击「发送验证邮件」，**Then** 发送验证码，输入后完成邮箱验证

---

### Story 5 — 我的订单（Priority: P1）

**Acceptance Scenarios**:
1. **Given** 已登录用户，**When** 进入「我的订单」，**Then** 展示该用户所有订单，按时间倒序，显示状态徽标
2. **Given** 用户点击订单卡片，**When** 跳转详情，**Then** 展示订单完整信息（课程/类型/年级/截止/微信/状态/报价）+ 状态时间线；管理员内部备注不对用户展示
3. **Given** 用户未登录，**When** 访问需登录页面，**Then** 跳转登录页，登录后返回原页面

---

### Story 6 — 提交需求（Priority: P1）

**提交字段**（微信号自动从用户资料带入，可修改）:
| 字段 | 类型 | 说明 |
|---|---|---|
| 课程名称 | 文本 | 必填 |
| 需求类型 | 下拉 | 必填，从管理后台获取 |
| 年级 | 下拉 | 必填，默认从用户资料带入 |
| 截止日期 | 日期 | 必填，须为未来日期 |
| 联系微信号 | 文本 | 必填，默认从用户资料带入 |

**Acceptance Scenarios**:
1. **Given** 已登录用户，**When** 提交需求，**Then** 生成唯一订单号，跳转成功页
2. **Given** 订单提交成功，**When** 数据库写入，**Then** Server酱 向管理员推送通知
3. **Given** 未登录用户点击「提交需求」，**When** 触发，**Then** 跳转登录页

---

### Story 7 — 活动公告（Priority: P2）

**Acceptance Scenarios**:
1. **Given** 用户进入活动页，**When** 页面加载，**Then** 展示所有活动，含已结束标签/剩余天数
2. **Given** 活动已过期，**When** 展示，**Then** 显示「已结束」标签但仍展示内容

---

### Story 8 — 论坛（Priority: P2）

论坛分两类帖子：**管理员公告**（仅管理员发布）和**用户讨论**（邮箱已验证用户发布，需管理员审核后公开）。

**帖子字段**：标题、正文（Markdown）、封面图URL（可选）、类型（ANNOUNCEMENT/DISCUSSION）

**Acceptance Scenarios**:
1. **Given** 管理员发布公告，**When** 前端刷新，**Then** 论坛列表出现该帖子（类型标签/标题/摘要/评论数）
2. **Given** 已验证用户发布讨论帖，**When** 提交，**Then** 帖子进入「待审核」状态，管理员审核通过后公开
3. **Given** 用户进入帖子详情，**When** 加载，**Then** 显示完整 Markdown 内容 + 评论列表
4. **Given** 已登录用户，**When** 提交评论，**Then** 评论显示在帖子下方（昵称+时间）
5. **Given** 未验证用户，**When** 尝试发帖或评论，**Then** 提示需要完成邮箱验证
6. **Given** 管理员在后台，**When** 隐藏/删除评论，**Then** 评论从前端消失

---

### Story 9 — 用户反馈（Priority: P2）

用户可提交问题反馈或改进建议，管理员查看并可回复，用户在「我的反馈」中看到历史和回复。

**反馈字段**：类型（BUG/建议/其他）、标题、详细描述

**Acceptance Scenarios**:
1. **Given** 已登录用户，**When** 提交反馈，**Then** 反馈保存成功，进入「我的反馈」列表（状态：待处理）
2. **Given** 管理员在后台，**When** 查看反馈列表，**Then** 展示所有用户反馈（类型/标题/提交人/时间/状态）
3. **Given** 管理员回复反馈，**When** 保存，**Then** 用户在「我的反馈」详情看到回复内容，状态更新为「已回复」
4. **Given** 管理员标记反馈为「已解决」，**When** 操作，**Then** 状态更新为「已解决」
5. **Given** 未登录用户，**When** 尝试提交反馈，**Then** 跳转登录页

---

### Story 10 — 历代作品轮播（Priority: P2）

管理员在后台上传/管理轮播条目，首页自动轮播展示。

**轮播字段**:
| 字段 | 说明 |
|---|---|
| 图片URL | 展示图片（管理员填入外链地址） |
| 课程名称 | 如「高等数学期末作业」 |
| 需求类型 | 如「期末作业」 |
| 完成时间 | 日期 |
| 好评语 | 用户评价摘要（管理员填写） |
| 订单号（可选） | 脱敏展示，如 JT-****-A3F2 |

**Acceptance Scenarios**:
1. **Given** 管理员添加轮播条目，**When** 首页刷新，**Then** 轮播区出现该条目，自动3秒切换
2. **Given** 多个轮播条目，**When** 展示，**Then** 按 sortOrder 排序，支持手动左右滑动

---

### Story 11 — 管理员接单与状态更新（Priority: P1）

**状态流转**:
```
PENDING（待确认）→ ACCEPTED（已接单）→ IN_PROGRESS（进行中）→ COMPLETED（已完成）
任意状态 → CLOSED（已关闭，终态）
```

**Acceptance Scenarios**:
1. **Given** 管理员登录深色后台，**When** 进入订单列表，**Then** 展示所有订单，状态汇总条 + 筛选 + 搜索
2. **Given** 管理员修改状态，**When** 保存，**Then** 状态更新 + Server酱 推送通知
3. **Given** 管理员填写报价，**When** 保存，**Then** 用户在订单详情可看到报价

---

### Story 12 — 需求类型与活动管理（Priority: P2）

同 v3 逻辑，管理员增删改需求类型（定价）和活动公告。

## UI 风格规范

### 用户端（浅色）
- 背景：`#f7f8fc`，卡片：`#ffffff`，圆角：`12px`
- 主色：`#1677ff`（蓝），强调：渐变 `#1677ff → #4096ff`
- 字体：系统默认 sans-serif，标题粗体
- 响应式断点：`768px`（手机），`1024px`（平板），`1280px`（桌面）
- 顶部导航：Logo + 菜单链接 + 用户头像/登录按钮，手机端收起为汉堡菜单
- 轮播：卡片式，自动3秒切换，支持触摸滑动

### 管理后台（深色）
- 背景：`#0f1117`，侧边栏：`#1a1d27`，卡片：`#1e2130`
- 主色：`#4f8ef7`，强调色：`#00d2ff`
- 左侧固定导航栏（宽220px），右侧内容区
- 数据卡片：带图标、数值大字、对比箭头
- 表格行 hover 高亮，状态标签彩色徽标

## 认证规范

- **注册**：用户名 + 昵称 + 邮箱（验证码确认）+ 密码（bcrypt hash）+ 手机 + 微信号 + 年级
- **登录**：邮箱 或 用户名 + 密码 → 返回 JWT（7天有效）
- **邮件服务**：Resend（发送验证码邮件）
- **验证码**：6位数字，5分钟有效，同邮箱60秒冷却
- **登录失败锁定**：连续5次失败，锁定5分钟

## Key Entities（更新）

- **User**：用户名、昵称、邮箱（已验证）、密码Hash、手机、微信号、年级、注册时间
- **EmailVerification**：邮箱验证码记录（邮箱、验证码、过期时间、是否已用）
- **OrderType**：需求类型定价
- **Activity**：活动公告
- **Order**：订单（关联 User）
- **StatusHistory**：状态变更记录
- **Post**：论坛帖子（管理员公告 or 用户讨论，含审核状态）
- **Comment**：评论（关联 Post + User，支持管理员隐藏）
- **Feedback**：用户反馈（私密，含类型/标题/描述/状态/管理员回复）
- **Carousel**：轮播条目（图片URL/课程名/类型/完成时间/好评语/排序）
- **Notification**：Server酱推送日志

## Success Criteria

- **SC-001**：用户从注册到成功提交需求，全程不超过 5 分钟
- **SC-002**：管理员新订单创建后 1 分钟内收到 Server酱 推送
- **SC-003**：所有页面在 iPhone 14 / Android 主流机型正常显示
- **SC-004**：首页轮播流畅，不闪烁，支持触摸滑动
- **SC-005**：论坛评论实时展示（提交后立即刷新）
