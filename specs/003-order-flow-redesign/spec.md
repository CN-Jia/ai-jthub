# Feature Specification: 订单流程重构 + 积分集成 + 论坛改版

**Feature Branch**: `003-order-flow-redesign`
**Created**: 2026-05-04 | **Status**: Clarified

---

## 一、概述

重构 JT-Hub 的核心订单流程，从「选购商品→下单」改为「直接提交需求→可选兑换服务→线下交易」模式。同时将积分系统与订单流程深度集成，并重新排版论坛为功能分区模式。

## 三、订单流程重构

### 3.1 新订单状态流转

```
已创建(CREATED)
  → 待接单(PENDING)
    → 进行中(IN_PROGRESS)  ← 管理员必须填写预计交付时间
      → 已结单(COMPLETED)  ← 自动发放奖励积分
任意状态 → 已取消(CANCELLED)  ← 终态
```

**状态说明**：
| 状态 | 说明 | 操作方 |
|---|---|---|
| CREATED | 用户刚提交，等待管理员确认 | 系统自动 |
| PENDING | 管理员已看到，待接单 | 管理员操作 |
| IN_PROGRESS | 已接单，进行中 | 管理员操作（必填交付时间） |
| COMPLETED | 已结单 | 管理员操作（自动发放积分） |
| CANCELLED | 已取消 | 用户或管理员 |

> **变更**：移除 `ACCEPTED` 状态（与 PENDING 合并），移除 `CLOSED`（改为 `CANCELLED`）。

### 3.2 提交需求页面重构（/submit）

**页面布局**：

```
┌─────────────────────────────────────────────┐
│  提交需求                                    │
│                                              │
│  课程名称 *    [____________________]         │
│  需求类型 *    [下拉选择____________]         │
│  年级 *        [下拉选择____________]         │
│  截止日期 *    [日期选择____________]         │
│  联系微信号    [自动带入可修改______]         │
│                                              │
│  ── 积分优惠（可选）──                        │
│  服务套餐/折扣券  [下拉：无 / 我的套餐列表]   │
│                                              │
│  ┌─ 折扣计算预览 ─┐                          │
│  │ 原价参考：200-500元                       │
│  │ 使用折扣券：8折                           │
│  │ 预估折后价：160-400元                     │
│  └────────────────┘                          │
│                                              │
│  ┌──────────────────────────────────────┐    │
│  │  💡 提交后请添加管理员微信            │    │
```

**下拉选项逻辑**：
1. 查询用户已兑换且未过期的服务套餐和折扣券
2. 服务套餐：显示名称 + 有效期（如「期中作业套餐 · 5月30日前有效」）
3. 折扣券：显示名称 + 折扣比例 + 有效期（如「8折优惠券 · 5月10日前有效」）
4. 选择折扣券后，根据需求类型的参考价格自动计算折后价并展示
5. 选择服务套餐后，标记该订单使用了哪个套餐

### 3.3 创建订单 API 变更

```json
// POST /api/orders
{
  "courseName": "高等数学",
  "orderTypeId": "...",
  "grade": "JUNIOR",
  "deadline": "2026-06-30T00:00:00Z",
  "contactWechat": "wxid_xxx",
  "source": "PC",
  // 新增：可选
  "redeemItemId": "...",     // 兑换的服务套餐/折扣券 ID
  "couponCode": "...",       // 或通过券码选择
}
```

**后端处理**：
1. 创建订单，状态为 `CREATED`
2. 如果传了 `redeemItemId`，验证该兑换项属于当前用户且未过期、未使用
3. 如果是折扣券，计算折后价记录到订单
4. 如果是服务套餐，标记订单关联该套餐
5. 标记兑换项为已使用
6. 返回订单号 + adminWechatId

### 3.4 订单详情页变更

**用户端**：
- 显示状态时间线
- 如果使用了折扣券/服务套餐，显示关联信息
- 进行中状态显示预计交付时间
- 已结单状态显示获得的奖励积分

**管理端**：
- 状态操作按钮：
  - `CREATED` → `PENDING`：「确认接单」
  - `PENDING` → `IN_PROGRESS`：「开始处理」（弹窗填写预计交付时间）
  - `IN_PROGRESS` → `COMPLETED`：「标记结单」（自动计算并发放奖励积分）
  - 任意 → `CANCELLED`：「取消订单」

### 3.5 奖励积分发放

**规则**：
- 订单标记为 `COMPLETED` 时自动发放
- 奖励数量 = 订单金额 × 对应订单类型的积分比例
- 比例在管理后台「需求类型」管理中配置（如：期中作业 5%，毕业设计 10%）
- 如果订单没有报价（quotedPrice），则按需求类型的参考价格中间值计算
- 管理员结单时可手动调整积分数量

**需求类型新增字段**：
```
pointRewardRate: Decimal  // 积分奖励比例，如 0.05 表示 5%
```

## 四、积分商城调整

### 4.1 兑换流程变更

**当前**：兑换 → 管理员审核 → 生效
**改为**：兑换 → 满足条件 → 直接生效（无需审核）

**自动生效条件**：
- 用户积分余额 >= 兑换所需积分
- 兑换项有库存（如有库存限制）
- 兑换项在有效期内

### 4.2 兑换项有效期

| 类型 | 有效期 | 说明 |
|---|---|---|
| 服务套餐 | 兑换后 30 天 | 过期自动作废 |
| 折扣券 | 兑换后 7 天 | 过期自动作废 |

### 4.3 兑换项在提交需求时展示

- 查询用户已兑换、未过期、未使用的兑换项
- 按类型分组展示（服务套餐 / 折扣券）
- 折扣券选择后自动计算折后价

## 五、论坛改版

### 5.1 功能分区

将论坛从「公告 + 讨论」双 tab 改为多板块模式：

| 板块 | 说明 | 谁能发帖 |
|---|---|---|
| 📢 公告通知 | 平台公告、活动通知 | 仅管理员 |

| ❓ 常见问题 | FAQ、使用教程 | 仅管理员 |
